STEM Foundations Portfolio Project

The Portfolio Project is a very unique project. Many schools don’t teach many 21st Century skills, but our school really values these skills. One important one is coding. So we are doing a big portfolio project. The Driving Question (DQ) for this project was "How can we share our ability to be reflective and innovative problem solvers with DRSS peers and community members?" So what is a portfolio? A portfolio is a unique & electronic way to share about ourselves, show off what we are learning at school, and much more. We coded our portfolios with HTML & CSS. Now what is HTML? HTML (Hyper Text Mark-up Language) is a fairly simple coding program that can help you create live websites on the Internet. What is CSS? CSS (Cascading Style Sheets) is a styling program that can really help you freely style your HTML document. Our portfolios have a CSS External Style Sheet that we use to style and we link it to our HTML documents. This project started in 6th Grade and ends in 12th. On our portfolios we have different required pages. These include an introduction to the portfolio (Homepage), a Coding Progression page which shows our growth in coding each year, a 5 Qualities Page (See 5 Qualities page) , a Landing Page which has highlighted projects from that year, a link for each previous year’s portfolios, a global CSS page, and more.

Reflective Commentary

a. The biggest struggle I had in this project was my styling. Especially in my External CSS. I had many selectors (selectors are the "tags" you use in CSS to style parts of your HTML document) overlap, so sometimes my content would look awkward (if you repeat selectors, the computer will look at the more accurate selector). Also, my background image (image on the back of my portfolio) wasn’t working for a while as a “saved image” (an image saved as a file to my computer). I overcame my CSS issues by using a HTML Validator (a code checker) and used a .jpg file extension (a .png file extension wasn’t working, and a .jpg extension is recommended over a .png).

c. Something I would have done differently in this project would be to write more CERs. What I mean is, my content for Class/Project pages' claims would be more convincing if I had used evidence. My content would also be more straightforward & easier to understand.

d. By undertaking this project at DRSS, I will be equipped with many 21st Century Skills when I walk out. Some of these skills include problem solving, critical thinking, analytical thinking, coding, thinking outside the box, critique/feedback, patience, all of the 5 Qualities, and writing skills. For example, I have become a better problem solver through the act of debugging frequently. After a numerous amount of times debugging, you know what questions to ask yourself when debugging.

My artifact is obviously my portfolio, which you are on right now. This shows my growth because I have improved in styling (as you can see from the comparison from my screenshotted Raptors Page & Candy Shoppe). I have borders (better ones), a nav bar, more images, a background image, and positioning of different paragraphs now. There is more on coding growth on my Coding Progression Page.
